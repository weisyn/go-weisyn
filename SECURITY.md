| 版本 | 支持状态          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## 报告漏洞

如果您发现安全漏洞，请**不要**公开报告。请通过以下方式私下报告：

1. **创建私有安全咨询**（推荐）：
   - 访问 [GitHub Security Advisory](https://github.com/weisyn/weisyn/security/advisories/new)
   - 创建新的安全咨询
   - 详细描述漏洞和复现步骤

2. **发送邮件**：
   - 发送至：wx@wesing.xyz

## 漏洞处理流程

1. **确认**：我们会在 48 小时内确认收到您的报告
2. **评估**：我们会在 7 天内评估漏洞的严重性
3. **修复**：我们会尽快修复漏洞
4. **披露**：修复后，我们会公开披露漏洞（在您同意的情况下）

## 安全最佳实践

使用 WES 区块链核心时，请遵循以下安全最佳实践：

- ✅ **密钥管理**：永远不要将私钥提交到代码仓库
- ✅ **网络安全**：在生产环境使用 TLS 加密连接
- ✅ **输入验证**：验证所有用户输入和交易数据
- ✅ **依赖更新**：定期更新依赖包以获取安全补丁
- ✅ **最小权限**：使用最小必要权限运行节点
- ✅ **共识安全**：确保节点配置正确，参与共识验证
- ✅ **智能合约审计**：部署智能合约前进行安全审计
