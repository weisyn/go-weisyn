## 🧱 资源级 vs 场景级

- **资源级示例**：
  - 模型：`models/examples/`（单个 ONNX 模型作为可执行资源）
  - 合约：`contracts/examples/`（单个 WASM 合约作为可执行资源）
  - 每个资源有自己的 README + 工件 + `testcases/default.json`，用于功能与回归测试。

- **场景级示例（本目录）**：
  - 聚焦「业务场景」而非单一资源，例如：
    - 代币转账场景（合约 + CLI）
    - DeFi 场景（合约 + SDK + 客户端）
    - AI+合约 场景（`models/examples` + `contracts/examples`）
  - 可以引用或依赖资源级示例，但职责是「讲清楚场景如何运作」。

---

## 🆚 与资源级示例的差异化价值

| 维度 | examples（场景级） | models/examples、contracts/examples（资源级） |
|------|-------------------|--------------------------------------------|
| **核心定位** | 完整业务场景示例 | 单个可执行资源（模型/合约） |
| **学习重点** | 如何组合资源构建应用 | 单个资源的功能与行为 |
| **内容形式** | 端到端场景演示（合约+客户端+脚本） | 固定行为的资源（.onnx/.wasm + testcases） |
| **目标用户** | 应用开发者、产品团队 | 资源开发者、平台验证者 |
| **技术栈** | 全栈应用开发 | 单一资源开发 |
| **测试方式** | 场景级集成测试 | 资源级单元测试 |
| **学习产出** | 能独立构建应用 | 能开发/验证单个资源 |

**关系说明**：
- 场景级示例可以**引用或依赖**资源级示例（例如：`examples/basic/token-transfer` 可以使用 `contracts/examples/simple-token`）
- 资源级示例专注于**单个资源的正确性**，场景级示例专注于**资源组合的业务价值**

## 🚀 快速开始

### 环境要求
- Go 1.24+
- TinyGo 0.31+（用于合约编译）
- Git
- WES 测试节点（运行示例需要）
- 支持的操作系统：Windows、macOS、Linux

### 一键体验
```bash
# 1. 启动 WES 测试节点
cd weisyn
./bin/testing

# 2. 在新终端中运行 hello-world 示例
cd examples/basic/hello-world
./scripts/build.sh      # 编译合约
./scripts/deploy.sh     # 部署合约
./scripts/interact.sh   # 与合约交互

# 3. 查看其他示例
cd ../token-transfer    # 架构演示（无需节点）
go run src/*.go         # 运行本地演示
./scripts/run_demo.sh
```

## 💡 最佳实践

### 学习建议
1. **循序渐进**: 按推荐路径学习，不要跳过基础
2. **动手实践**: 每个示例都要亲自运行和修改
3. **深入理解**: 阅读详细注释，理解设计思路
4. **举一反三**: 基于示例开发自己的应用

### 开发建议
1. **代码规范**: 遵循示例中的编码标准
2. **安全第一**: 重视数据加密和权限控制
3. **错误处理**: 完善的异常处理和用户提示
4. **测试驱动**: 编写充分的测试用例

### 部署建议
1. **环境隔离**: 开发、测试、生产环境分离
2. **配置管理**: 使用配置文件管理环境差异
3. **监控日志**: 完善的日志记录和监控
4. **备份策略**: 重要数据的备份和恢复方案
