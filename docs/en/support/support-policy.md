# Support Policy

---

## Overview

This document defines WES version support policies and lifecycle.

---

## Version Types

| Type | Description | Support Period |
|------|-------------|----------------|
| LTS | Long-Term Support version | 2 years |
| Current | Current version | Until next version release |
| Preview | Preview version | No support provided |

---

## Support Levels

### Full Support

- Bug fixes
- Security updates
- Performance improvements
- Documentation updates

### Security Support

- Security updates only
- Critical bug fixes

### No Support

- No longer receive any updates
- Upgrade recommended

---

## Version Lifecycle

```
Release → Full Support → Security Support → EOL (End of Life)
```

### LTS Version Timeline

| Phase | Duration |
|-------|----------|
| Full Support | 12 months |
| Security Support | 12 months |
| Total | 24 months |

### Current Version Timeline

| Phase | Duration |
|-------|----------|
| Full Support | Until next version release |
| Security Support | 3 months |

---

## Upgrade Recommendations

### When to Upgrade

- When new LTS version is released
- When current version enters security support period
- When new features are needed

### Upgrade Path

- Support upgrading across one major version
- Major version upgrades require migration guide reference
- Recommend testing in test environment first

---

## Security Response

### Security Vulnerability Levels

| Level | Response Time |
|-------|---------------|
| Critical | Within 24 hours |
| High | Within 72 hours |
| Medium | Within 7 days |
| Low | Next version |

### Security Update Notifications

- Mailing list
- GitHub Security Advisories
- Official blog
