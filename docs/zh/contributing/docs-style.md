# 文档规范

---

## 概述

本文档定义了 WES 项目的文档编写标准。

---

## 文档组织

### 目录结构

```
docs/zh/
├── getting-started/   # 入门指南
├── concepts/          # 核心概念
├── tutorials/         # 教程
├── how-to/            # 操作指南
├── reference/         # 参考文档
├── contributing/      # 贡献指南
└── support/           # 支持信息
```

### 文档类型

| 类型 | 用途 | 特点 |
|------|------|------|
| 概念文档 | 解释是什么、为什么 | 理论性、背景知识 |
| 教程 | 手把手教学 | 步骤明确、可跟随 |
| 操作指南 | 完成特定任务 | 目标导向、实用 |
| 参考文档 | 详细的技术规格 | 全面、准确 |

## 写作规范

### 标题

- 使用 `#` 作为一级标题
- 标题层级不超过 4 级
- 标题应简洁明确

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题
```

### 段落

- 段落之间空一行
- 每段表达一个主要观点
- 保持段落简短

### 列表

无序列表：
```markdown
- 项目 1
- 项目 2
- 项目 3
```

有序列表：
```markdown
1. 步骤 1
2. 步骤 2
3. 步骤 3
```

### 代码

行内代码：
```markdown
使用 `wes-node start` 启动节点。
```

代码块：
````markdown
```go
func main() {
    fmt.Println("Hello, WES!")
}
```
````

### 表格

```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 值 1 | 值 2 | 值 3 |
```

### 链接

内部链接：
```markdown
[安装指南](./installation.md)
[核心概念](../concepts/)
```

外部链接：
```markdown
[GitHub](https://github.com/weisyn/weisyn)
```

## 风格指南

### 语言

- 使用简体中文
- 技术术语保持一致
- 避免口语化表达

### 语气

- 使用第二人称（"你"）
- 保持专业但友好
- 避免过于正式

### 术语

- 首次出现时解释术语
- 使用术语表中的标准术语
- 英文术语保持原样或附注中文
